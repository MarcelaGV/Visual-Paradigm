Use Case Note
■ Workflow
    • // Write down briefly how user perform the work
    • El administrador selecciona un servicio.
    • Modifica precio, descripción, duración o categoria.
    • Guarda los cambios.
    • El sistema confirma la actualización.
■ Business Logic
    • // Write down what user expect the system to react upon certain condition (e.g. low inventory alert level)
    • El precio debe ser mayor a cero.
    • Solo usuarios con rol administrador pueden modificar servicios.
■ Decisions
    • // Write down the decisions made during the meeting (e.g. Must allow accessing from mobile devices)
    • Se acordó que solo el administrador puede editar servicios.
    • Se decidió que el historial de cambios no será visible al cliente (por ahora).
■ Follow-up
    • // Write down the items that should follow-up in the coming meeting
    • Definir si se guardará historial de modificaciones.
    • Evaluar notificar cambios importantes a los usuarios.

Administrador
Steps
1. El usuario igresa el URL o ingresa por medio de LINK de Naiapp
2. El administrador inicia sesión con sus credenciales en la sección "Soy parte de NaiApp"
3. Administrador ingresa a la opción “Servicios”.
4. El sistema consulta la base de datos.
5. El usuario selecciona servicio del listado.
6. Hace clic en botón “Editar” (icono de lapiz).
7. El sistema habilita campos.
8. Modifica precio, duración o categoria.
9. if Quedo algun campo quedo sin llenar
    9.1. El sistema muestra mensaje indicando los campos faltantes.
   end if
10. Hace clic en “Guardar cambios”.
11. El sistema valida datos.
12. Actualiza base de datos.
