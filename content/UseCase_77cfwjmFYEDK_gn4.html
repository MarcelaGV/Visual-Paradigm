Use Case Note
■ Workflow
    • // Write down briefly how user perform the work
    • El sistema verifica las franjas horarias disponibles del profesional
    • El sistema verifica las citas previamente registradas
    • El usuario selecciona fecha y profesional
    • Valida que no exista otra cita registrada en el mismo horario
■ Business Logic
    • // Write down what user expect the system to react upon certain condition (e.g. low inventory alert level)
    • El sistema debe considerar la duración del servicio para validar disponibilidad
    • El sistema debe actualizar en tiempo real los horarios disponibles despues de registrar una nueva cita
    • Solo el administrador podrá modificar manualmente la disponibilidad del profesional

Scenario
Steps
1. El usuario selecciona Agendar cita
2. El usuario selecciona fecha y profesional
3. El sistema consulta la agenda del profesional
4. El sistema verifica las citas previamente registradas
5. El sistema valida la duración del servicio seleccionado
6. El sistema muestra las franjas horarias disponibles
7. El sistema permite seguir con el proceso de agendamiento
