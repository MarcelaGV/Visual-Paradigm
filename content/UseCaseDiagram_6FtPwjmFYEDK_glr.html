Model Elements
Name Description
  Sistema NaiApp : System
  Cliente : Actor
  Administrador : Actor
  Profesional : Actor
  Agendar cita : Use Case

El sistema permite al cliente o administrador crear una nueva cita seleccionando fecha, hora, servicio y profesional disponible

  Validar disponibilidad : Use Case

El sistema verifica franjas de horarios disponibles para el agendamiento de citas de los empleados

  Consultar agenda de citas : Use Case

 El sistema permitirá visualizar la agenda de citas mediante un calendario filtrada por día o profesional

  Registrar nuevos servicios : Use Case

El sistema permitirá al administrador registrar nuevos servicios (ej. Manicura, Masaje) especificando nombre, descripción, duración y precio

  Registro de Usuarios : Use Case

El sistema debe permitir el registro de usuarios con sus respectivos roles (como clientes y empleados), mediante sus credenciales (correo electrónico y contraseña) y el sistema guarda la información en la base de datos

  Actualizar información de perfil : Use Case

El sistema permitirá actualizar la información del perfil del usuario (teléfono, contraseña, etc)

  Inhabilitar cuenta de usuario : Use Case

 El sistema permite realizar la desactivación lógica de una cuenta de usuario cuando este lo solicite o por decisión administrativa

  Buscar perfil de usuario : Use Case

El sistema permitirá buscar y visualizar el perfil de un usuario mediante filtros como nombre, identificación o correo electrónico

  Reprogramar cita : Use Case

El sistema permitirá reprogramar una cita (cambio de horario o especialista) notificando automáticamente a las partes involucradas

  Cancelar cita programada : Use Case

El sistema permitirá cancelar una cita programada, liberando el espacio en el calendario de disponibilidad

  Buscar servicios : Use Case

El sistema permitirá a los clientes navegar y buscar servicios por categorías o palabras clave.

  Actualizar detalles de servicios : Use Case

El sistema permite actualizar los detalles de los servicios, como precio o cambios en la duración del procedimiento.

  Eliminar servicios del catálogo : Use Case

El sistema permite eliminar servicios del catálogo para que no aparezcan en el agendamiento.

  Generar nuevos reportes de pagos : Use Case

El sistema permite generar reportes de pagos a partir de criterios definidos (día/semana/mes/profesional/servicio)

  Visualizar reportes generados de pagos : Use Case

El sistema permitirá visualizar los reportes generados en pantalla con gráfica de los ingresos en el período seleccionado y tabla de datos

  Editar parámetros de reportes generados de pagos : Use Case

El sistema permite editar los parámetros de un reporte ya configurado (ej. cambiar el rango de fechas antes de exportar)

  Borrar historial de reportes de pagos generados : Use Case

El sistema permite eliminar registros de reportes de pagos guardados o históricos del panel administrativo para limpieza de datos

  Registrar nuevo pago : Use Case

El sistema permite al administrador registrar el medio de pago que utilizan los clientes

  Consultar historial de transacciones : Use Case

El sistema permite consultar el historial de transacciones y el estado de los pagos realizados por cada cliente.

  Editar registro de pago : Use Case

El sistema permite modificar la información de un pago registrado para corregir errores (ej. si se marcó efectivo pero fue transferencia) antes del cierre de caja

  Anular registro de pago : Use Case

El sistema permite anular un pago previamente registrado en caso de errores graves o devoluciones, dejando trazabilidad del motivo.

  Inicio de sesión : Use Case

El sistema permite a los usuarios autenticarse mediante credenciales, validando su información y otorgando acceso según su rol.

  Registrar calificación de servicio : Use Case

El sistema permite al cliente registrar una calificación sobre un servicio recibido, validando que la cita haya sido completada y almacenando la información.

  Visualizar promedio de calificaciones : Use Case

El sistema calcula y muestra el promedio de calificaciones de los profesionales, los servicios y el establecimiento

  Validar datos : Use Case

El sistema verifica que las credenciales ingresadas por parte del usuario sean correctas

  Calendario profesional : Use Case

El sistema muestra la disponibilidad del profesional en un calendario, permitiendo visualizar y gestionar los horarios de atención

  Asignar servicios : Use Case

El sistema permite seleccionar uno o varios servicios para asociarlos a una cita, calculando la duración total y el costo correspondiente

  Crear orden : Use Case

El sistema permite al usuario con permisos editar una orden existente, agregando o eliminando servicios, recalculando automáticamente el valor total y actualizando la información almacenada

  Modificar orden : Use Case

El sistema permite al usuario con permisos editar una orden existente, agregando o eliminando servicios, recalculando automáticamente el valor total y actualizando la información almacenada
